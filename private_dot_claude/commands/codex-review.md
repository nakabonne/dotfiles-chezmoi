# Codex Review

OpenAI Codex によるコードレビューを `codex-reviewer` サブエージェントに委譲する。Codex の出力でこの会話のコンテキストを汚染しないために、必ずサブエージェント経由で実行する。

## あなたのタスク

`Agent` ツールを使い、`subagent_type: "codex-reviewer"` でサブエージェントを起動する。

サブエージェントへ渡すプロンプトは次のように組み立てる:

- **`$ARGUMENTS` が空でない場合**: その内容をレビュー範囲として渡す。例:
  - `go/services/foo/handler.go` → そのファイルだけをレビュー
  - `請求書の計算ロジック` → その機能に関連する箇所をレビュー
  - `last commit` → working tree ではなく HEAD コミットをレビュー
- **`$ARGUMENTS` が空の場合**: 現在の未コミット変更（`git status` + `git diff`）をレビュー対象とするよう指示する。未コミット変更が無ければ最新コミットにフォールバックさせる

プロンプトには必ず以下を含める:
- このレビューはユーザーが `/codex-review` で明示的にトリガーしたものであること（オンデマンドの依頼）
- サブエージェントは標準の手順に従うこと: `codex exec --sandbox read-only` を実行し、`REVIEW.md` を尊重し、Codex の出力を日本語のままそのまま返す
- サブエージェントは Codex の出力に自分の意見を上乗せしないこと

## サブエージェントの戻り値を受け取った後

- サブエージェントの出力（Codex レビュー本文を含む）をそのままユーザーに提示する
- 再レビューや論評を加えない。Codex の指摘を読んでどうするかはユーザーが決める
- Codex が「重大な指摘」を出していた場合のみ、対応するか確認するために 1 行だけユーザーに問いかける
